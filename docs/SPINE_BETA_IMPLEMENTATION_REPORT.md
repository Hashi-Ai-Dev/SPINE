## Issue #65 ‚Äî `check before-pr --json` structured doctor detail

**Date:** 2026-04-09
**Branch:** `beta/usability65-before-pr-json-structure`
**Status:** ‚úÖ Complete ‚Äî pending PR

### Problem

`spine check before-pr --json` existed but the per-check-item JSON output lacked:
- A `category` field to classify what domain each check covers
- Structured `detail` for the `doctor` check ‚Äî errors and warnings were collapsed into a prose `message` string

Agents and automation tools could not reliably parse which doctor sub-issues triggered a check failure.

### Solution

Minimal backward-compatible extension to `CheckItem` and the JSON serializer:

**`CheckItem` ‚Äî new fields:**
- `category: str` ‚Äî domain classification for each check (`structure`, `config`, `health`, `history`, `context`)
- `detail: list[dict] | None` ‚Äî structured sub-items, currently only populated by the `doctor` check

**Per-check categories:**
| Check name     | Category    |
|----------------|-------------|
| `spine_dir`    | `structure` |
| `mission`      | `config`    |
| `doctor`       | `health`    |
| `drift`        | `history`   |
| `evidence`     | `history`   |
| `decisions`    | `history`   |
| `branch_context` | `context` |
| `recent_brief` | `context`   |

**Doctor `detail` shape** (present only when doctor has issues):
```json
[
  {"severity": "error|warning", "file": "<path>", "message": "<text>"}
]
```

### Exact JSON shape added

Before (existing fields, unchanged):
```json
{
  "result": "pass|review_recommended",
  "passed": true,
  "repo": "/path/to/repo",
  "branch": "main",
  "default_branch": "main",
  "checked_at": "2026-04-09T...",
  "checks": [
    {"name": "doctor", "status": "pass", "message": "repo health OK"}
  ]
}
```

After (new fields added):
```json
{
  "result": "pass|review_recommended",
  "passed": true,
  "repo": "/path/to/repo",
  "branch": "main",
  "default_branch": "main",
  "checked_at": "2026-04-09T...",
  "checks": [
    {
      "name": "spine_dir",
      "status": "pass",
      "message": ".spine/ found",
      "category": "structure"
    },
    {
      "name": "doctor",
      "status": "fail",
      "message": "1 doctor error(s): .spine/mission.yaml: YAML parse error: ...",
      "category": "health",
      "detail": [
        {"severity": "error", "file": ".spine/mission.yaml", "message": "YAML parse error: ..."}
      ]
    }
  ]
}
```

Note: `detail` is absent (not `null`) when the doctor check has no issues.
