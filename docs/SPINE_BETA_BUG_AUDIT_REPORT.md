## 3. High-Priority Summary

### Blockers

None that block the CLI from functioning. However:

- **A-01 + A-02 together make the `hooks` feature nearly unusable in practice.** `spine hooks install` installs a hook that will (a) fail with "command not found" because it uses `spine` not `uv run spine`, and (b) if `spine` were in PATH, would always return exit 1 on normal healthy repos due to subdirectory warnings. Both bugs were introduced together in PR #41 (Issue #34).

### High-Severity Issues

| ID | Issue |
|---|---|
| A-01 | `check before-pr` fails on healthy repos — subdirectory warnings cause false exit 1 |
| A-02 | Hook script uses `spine` without `uv run` — hook fails with "command not found" |
| A-06 | AGENTS.md template has two invalid commands (`brief generate`, `opportunity add`) |
| A-08 | `SPINE_STATUS.md` shows done work as pending, has a corrupted table row |

### Anything That Should Interrupt the Current Feature Queue

Yes. **A-01 + A-02 should be fixed before Issue #36 (mission refine draft flow) proceeds.** The hooks feature was the last Beta slice before #36. If A-01/A-02 are not addressed, the just-shipped hooks feature (#34) is non-functional in standard setups — and `check before-pr` is itself unreliable as a governance gate. Shipping #36 on top of a broken `check before-pr` deepens the inconsistency.

**A-06** (AGENTS.md template bad commands) should also be fixed — it directly contaminates governed repos that run `spine init`.
